# ATENÇÃO LEIA-ME!!!
## Neste repositório devem ser armazenados documentos relativos ao projeto, por
exemplo:
* casos de teste
* Plano de Gerenciamento do Projeto
* História de Usuário
* Modelo de Dados Conceitual etc

## Para sistemas que NÃO TRABALHAM COM GDF a seguinte estrutura de pastas
deve ser criada no branch master:
* engenharia-de-software - Essa pasta armazenará documentos relativos a
especificação, desenvolvimento, manutenção de sistemas de software, como por
exemplo: especificação de requisitos, diagramas, modelo entidade-relacionamento,
protótipo de telas, pareceres técnicos, plano de treinamento etc.
* operacao - Essa pasta guardará documentos de ordem operacional, como por
exemplo: rotinas de produção, scripts de produção, instalação de sistemas etc.
* qualidade - Essa pasta armazenará documentos relacionados a gestão de
qualidade de software, como por exemplo: scripts de teste, evidências de execução
de testes, entre outros.
